# Checklists del proyecto

Esta carpeta contiene checklists practicas para revisar tareas antes, durante y despues de implementarlas.

Las checklists no sustituyen las instrucciones especificas del Director Tecnico ni del Arquitecto Senior; sirven como apoyo para no olvidar verificaciones importantes.

## Cuando usar cada checklist

- `CHECKLIST_BEFORE_IMPLEMENTATION.md`: antes de iniciar cualquier tarea.
- `CHECKLIST_BEFORE_COMMIT.md`: antes de preparar commit o entregar resultado.
- `CHECKLIST_DATABASE_CHANGE.md`: cuando se toquen migraciones, RLS, RPCs, triggers, constraints o tipos.
- `CHECKLIST_PUBLIC_ROUTE_SECURITY.md`: cuando se toquen rutas publicas como `/solicitud` o `/estado`.
- `CHECKLIST_INTERNAL_UI_QA.md`: cuando se toquen pantallas internas, formularios o UI protegida.
- `CHECKLIST_PHASE_CLOSURE.md`: al cerrar una fase completa.
